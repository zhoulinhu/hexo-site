多博客搭配文档仓库的搭建

John Doe2026-06-112026-06-12

思路介绍

问题来源: 原来搭建jekyll搭配Chirpy主题后，在里面_posts文件夹下编写文档然后推送发布到仓库，仓库Actions自动编译构建发布Github Pages，这没问题
突发奇想，我如果想在github pages搭建多个不同的博客系统喃，比如再用hexo搭建一套，而文档还是那些文档，不可能在把文档复制一份到hexo项目的source/_posts文件下吧，每次编写一篇文档我都要复制一次，那就太麻烦了，所以有了文档和项目分离的想法。
那如何去做喃，根据搜索了解，思路大致那么几个：

  1. 专门一个文档仓库blog-posts(用来管理文档)，每次编写修改这个文档仓库并推送，github actions向另外的博客仓库发送信号，另外的博客仓库来拉取跟新内容重新构建
  2. blog-posts仓库更新，github actions直接推送_posts文件下内容到博客仓库对应位置，博客仓库actions触发更新重新构建
  3. 将blog-posts仓库设置成博客仓库的子模块，blog-posts更新后，博客仓库重新拉取子模块push触发actions重新构建。
    {: .prompt-tip}

原本打算实现思路1的，但是又突然想到想把博客仓库设置成私有的，不想让别人可以直接访问仓库获取文档源文件，捣鼓半天发现Github免费版账号，想要使用Github Pages必须吧这个账号设置成public的，原本的前两个思路就不是那么适用了，有点麻烦还不好实现，最终还是选择把blog-posts设置成博客仓库的子模块可能要好些。
{: .prompt-info}

思路1的实现

这种思路是一种CI/CD自动化中的一个标准模式
核心思路是：将文章仓库blog-posts作为一个“触发器”，每次推送更新时，自动向两个博客仓库(jekyll-blog,hexo-blog)发送“远程唤醒信号”
这个远程唤醒方式主要有两种：respository_dispatch和workflow_dispatch
{: .prompt-info}

方案 A：使用 repository_dispatch (推荐)

这是官方推荐的，文章仓库扮演”消息发布者”, 博客仓库扮演”监听者”，一有新文章就自动构建
{: .prompt-info}

  1. 在博客仓库中创建监听器
    在每个博客仓库(jekyll-blog和hexo-blog)的.github/workflows/目录下，创建一个YAML配置文件(例如deploy-on-update.yml)
    这个监听工作流的重点，是让它响应on: repository_dispatch事件
点击展开详细的YAML文件内容 

# .github/workflows/deploy-on-update.yml (在 jekyll-blog 和 hexo-blog 仓库中各放一份)
name: Deploy on Blog Update

# 监听 repository_dispatch 事件，并且只关心 event-type 为 'blog-updated' 的信号
on:
  repository_dispatch:
    types: [blog-updated]

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout 仓库代码
        uses: actions/checkout@v4
        with:
          submodules: 'recursive'  # 务必加上，这样才能拉取最新的文章子模块

      - name: 设置 Node.js (用于 Hexo)
        uses: actions/setup-node@v4
        with:
          node-version: '18'

      - name: 安装依赖 (如果是 Jekyll 则需要 Ruby 环境)
        # 这里假设你的博客根目录有 package.json
        run: npm install

      - name: 构建博客
        run: npm run build # 或者你对应的构建命令，如 jekyll build

      - name: 部署到 GitHub Pages
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./public # 根据你的博客生成目录调整



2. 在文章仓库中创建触发器  
    在文章仓库`blog-posts`的`.github/workflows/`目录下，创建一个触发器工作流(例如`tigger-blogs.yml`)。它的任务很简单：当文章更新时，通知博客仓库。



点击展开详细的YAML文件内容



# .github/workflows/trigger-blogs.yml (在 my-blog-posts 仓库中)
name: Trigger Blog Builds

# 当文章被推送到 main 分支时触发
on:
  push:
    branches: [ "main" ]
    paths: # 优化：仅当文章目录下有变化时才触发
      - '_posts/**'
      - '**.md'

jobs:
  trigger-builds:
    runs-on: ubuntu-latest
    steps:
      # 使用官方 action 来发送 repository_dispatch 信号
      - name: 触发 Jekyll 博客构建
        uses: peter-evans/repository-dispatch@v3
        with:
          token: ${{ secrets.PAT_TOKEN }} # 关键！使用 PAT
          repository: your-username/jekyll-blog # 你的 Jekyll 博客仓库
          event-type: blog-updated           # 与监听器中定义的 type 一致
          client-payload: '{"ref": "${{ github.ref }}", "sha": "${{ github.sha }}"}' # 可选，传递更多信息

      - name: 触发 Hexo 博客构建
        uses: peter-evans/repository-dispatch@v3
        with:
          token: ${{ secrets.PAT_TOKEN }}
          repository: your-username/hexo-blog
          event-type: blog-updated
          client-payload: '{"ref": "${{ github.ref }}", "sha": "${{ github.sha }}"}'[reference:0]
